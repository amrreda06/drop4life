## التثبيت والإعداد

### س1: كيف أطبق Migration؟
**ج:** اتبع هذه الخطوات:
```bash
cd c:\Users\amrreda\Desktop\New folder
python manage.py migrate
```

إذا نجحت، ستظهر: `Operations to perform: ... OK`

### س2: ماذا لو حدث خطأ في Migration؟
**ج:** جرّب هذا:
```bash
# عرض جميع الـ Migrations
python manage.py showmigrations api

# تطبيق مجموعة محددة
python manage.py migrate api 0010

# إذا فشل، قد تحتاج لحذف الملف
del db.sqlite3
python manage.py migrate
```

### س3: هل أحتاج لإعادة تشغيل الخادم؟
**ج:** نعم، **يجب** إعادة تشغيل الخادم بعد Migration:
```bash
# اضغط Ctrl+C لإيقاف الخادم
# ثم اكتب:
python manage.py runserver
```

### س4: لا أرى الزر في الجدول
**ج:** تحقق من:
- ✅ هل قمت بـ Migration؟
- ✅ هل الخادم معاد تشغيله؟
- ✅ هل صفحة المتصفح معاد تحميلها (Ctrl+Shift+R)؟
- ✅ هل المستخدم له صلاحيات (معمل أو سوبر أدمن)؟

## الاستخدام والعمليات

### س8: ما الفرق بين الفصيلة المبدئية والمؤكدة؟
**ج:**
- **المبدئية:** الفصيلة المسجلة عند تبرع الدم
- **المؤكدة:** الفصيلة التي يؤكدها المعمل (قد تكون مختلفة!)

### س9: كيف أقبل عينة؟
**ج:** 
1. اختر فصيلة دم من القائمة
2. يمكنك اختيار أمراض (اختياري)
3. اضغط "✅ قبول العينة"
4. الكيس سيضاف للمخزون فوراً

### س10: كيف أرفض عينة؟
**ج:**
1. اختر الأمراض المكتشفة (اختياري)
2. **يجب** أن تدخل سبب الرفض
3. اضغط "❌ رفض العينة"
4. إشعار سيرسل للأدمن

### س11: ماذا لو لم أختر فصيلة عند القبول؟
**ج:** ستظهر رسالة خطأ:
```
⚠️ يرجى اختيار فصيلة الدم المؤكدة
```

### س12: ماذا لو لم أدخل سبب عند الرفض؟
**ج:** ستظهر رسالة خطأ:
```
⚠️ يرجى إدخال سبب الرفض
```

## المخزون والإشعارات

### س13: هل يضاف الكيس للمخزون تلقائياً؟
**ج:** نعم! عند القبول:
- ✅ الكيس يضاف للمخزون مباشرة
- ✅ الفصيلة المؤكدة تُستخدم
- ✅ العدد يتحدّث فوراً

### س14: من يستقبل الإشعارات عند الرفض؟
**ج:** جميع **الأدمن** في النظام:
- 👔 السوبر أدمن (DR)
- 👤 الأدمن (ADM)

### س15: ماذا يحتوي الإشعار؟
**ج:** يحتوي على:
- 🔴 رقم الكيس
- 💊 الأمراض المكتشفة
- 📝 سبب الرفض
- 👤 اسم المحلل

### س16: هل يمكن إعادة تحليل الكيس بعد الرفض؟
**ج:** لا، مرة واحدة فقط. الحالات:
- **Pending** → تحليل ممكن
- **Approved** ✅ → تحليل مكتمل
- **Rejected** ❌ → لا يمكن إعادة التحليل

## قاعدة البيانات

### س17: أين يتم حفظ البيانات؟
**ج:** في جدول جديد اسمه `api_sampleanalysis`:

```sql

```

### س18: كيف أعرض جميع التحليلات؟
**ج:** استخدم API:
```bash
curl http://localhost:8000/api/sample-analyses/
```

### س19: كيف أحذف تحليل؟
**ج:** لا يمكن حذف التحليلات (مهم للأمان)!
- ✅ يمكن مراجعة التاريخ
- ✅ كل عملية مسجلة
- ❌ لا يمكن الحذف

### س20: هل يمكن تحديث التحليل بعد القبول؟
**ج:** لا، مرة واحدة فقط:
- بعد `POST` (الإنشاء)
- تحديث واحد فقط `PUT` (القبول أو الرفض)
- لا يمكن التغيير بعد ذلك

**ج:** تحقق من:
- ✅ هل الرفض تم بنجاح (404 vs 200)
- ✅ هل المستخدم تسجيل دخول؟
- ✅ الـ Logs: `python manage.py logs`

## الحالات الخاصة

### س36: ماذا لو دخل مستخدمان في نفس الوقت؟
**ج:** لا مشكلة، يمكنهما:
- ✅ تحليل أكياس مختلفة
- ✅ عزل البيانات (Transactions)
- ✅ كل واحد يعمل مستقل

### س37: هل يمكن تحليل نفس الكيس مرتين؟
**ج:** لا، فقط مرة واحدة:
- بعد الإنشاء (Status = Pending)
- تحديث واحد فقط (Approved أو Rejected)
- بعدها Fixed

### س38: ماذا لو غيرت الفصيلة المبدئية للكيس؟
**ج:** لا يؤثر، لأن:
- الفصيلة المؤكدة منفصلة
- التحليل استقلالي
- الكيس ينضم للمخزون بـ الفصيلة المؤكدة

### س39: هل يمكن تغيير الأمراض بعد الحفظ؟
**ج:** لا، البيانات مقفلة عند `Created`:
- ✅ القراءة ممكنة

- ✅ العرض في الإشعارات
