# دليل استخدام نظام تحليل العينات

## نظرة عامة
تم إضافة نظام متكامل لتحليل عينات الدم في وحدة المعمل. يمكن للموظفين المصرح لهم تحليل العينات واختيار فصيلة الدم والأمراض المكتشفة ثم قبول أو رفض العينة.

## الوصول إلى النظام

### متطلبات الوصول:
- المعمل (Lab - MLS)
- نائب المدير (Deputy - DPM) 
- السوبر أدمن (Super Admin - DR)

### رابط الوصول:
```
/api/sample-analyses/
```

## سير العمل

### الخطوة 1: عرض العينات المعلقة
1. ادخل إلى وحدة الفحص المعملي (🔬)
2. سترى جدول بجميع الأكياس المعلقة للفحص
3. يظهر في الجدول:
   - رقم الكيس
   - اسم المتبرع
   - الفصيلة المبدئية
   - حالة الفحص
   - التاريخ
   - زر "تحليل" لكل كيس

### الخطوة 2: فتح نافذة التحليل
1. اضغط على زر "🔬 تحليل" للكيس المطلوب
2. ستظهر نافذة (Modal) تحتوي على:
   - معلومات الكيس (للعرض فقط)
   - قائمة لاختيار الفصيلة المؤكدة
   - قائمة الأمراض (Checkboxes)
   - حقل سبب الرفض (يظهر عند الحاجة)
   - زر القبول والرفض

### الخطوة 3: اختيار الفصيلة المؤكدة
1. اختر من القائمة المنسدلة:
   - O+
   - O-
   - A+
   - A-
   - B+
   - B-
   - AB+
   - AB-

### الخطوة 4: اختيار الأمراض المكتشفة (إن وجدت)
اختر واحد أو أكثر من:
- **HIV**: فيروس نقص المناعة
- **Hepatitis B**: التهاب الكبد الفيروسي ب
- **Hepatitis C**: التهاب الكبد الفيروسي ج
- **Syphilis**: الزهري
- **Malaria**: الملاريا
- **Contaminated**: عينة ملوثة
- **Expired**: عينة منتهية الصلاحية
- **Other**: أخرى

### الخطوة 5: قبول العينة ✅

**الشروط:**
- يجب اختيار فصيلة الدم المؤكدة

**النتائج:**
1. تحديث حالة الكيس إلى "Approved"
2. إضافة الكيس إلى مخزون الدم
3. إرسال إشعار تأكيد
4. تسجيل العملية في سجل التدقيق

### الخطوة 6: رفض العينة ❌

**المتطلبات:**
- يجب إدخال سبب الرفض
- الأمراض المكتشفة اختيارية

**النتائج:**
1. تحديث حالة الكيس إلى "Rejected"
2. حفظ الأمراض المكتشفة والسبب
3. إرسال إشعار للأدمن يتضمن:
   - رقم الكيس المرفوض
   - الأمراض المكتشفة
   - سبب الرفض
4. تسجيل العملية في سجل التدقيق

## API التفاصيل

### الحصول على جميع التحليلات
```
GET /api/sample-analyses/
```

### الحصول على تحليل محدد
```
GET /api/sample-analyses/{bag_id}/
```

### إنشاء تحليل جديد
```
POST /api/sample-analyses/
Content-Type: application/json

{
  "bagId": "BAG-001",
  "confirmedBloodType": "O+",
  "detectedDiseases": ["HIV"],
  "status": "Pending"
}
```

### تحديث التحليل (القبول)
```
PUT /api/sample-analyses/{bag_id}/
Content-Type: application/json

{
  "confirmedBloodType": "O+",
  "detectedDiseases": [],
  "status": "Approved"
}
```

### تحديث التحليل (الرفض)
```
PUT /api/sample-analyses/{bag_id}/
Content-Type: application/json

{
  "detectedDiseases": ["HIV", "Hepatitis B"],
  "rejectionReason": "فيروسات خطيرة مكتشفة",
  "status": "Rejected"
}
```

## معلومات الإشعارات

### عند القبول:
- ✅ رسالة نجاح
- الكيس أضيف للمخزون

### عند الرفض:
- ❌ رسالة رفض
- تصل إلى جميع المسؤولين
- تتضمن:
  - رقم الكيس
  - الأمراض المكتشفة
  - السبب الطبي

## سجل التدقيق (Audit Log)
جميع العمليات تسجل تلقائياً:
- معرف المستخدم
- الصلاحية
- نوع العملية
- التفاصيل الكاملة

## ملاحظات مهمة

1. **الصلاحيات**: تم تقيد الوصول للمعمل والنائب والسوبر أدمن فقط
2. **البيانات**: جميع البيانات محفوظة في قاعدة البيانات
3. **الإشعارات**: يتم إرسال إشعارات فورية عند الرفض
4. **التحديثات**: المخزون يتحدث فوراً عند قبول العينة
